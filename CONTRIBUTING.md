# Contribution Guidelines

Thank you for your interest in contributing to the Hostweb project! Following these guidelines will help make the process smoother for everyone involved.

## How to Contribute
1. **Check for Issues:** Before starting, make sure there isn't an existing issue equivalent to your intended contribution.
2. **Fork the Repository:** Create a personal copy of the repository on GitHub.
3. **Create a Branch:** Work on a new branch with a descriptive name related to your changes.
4. **Make Changes:** Implement your changes and commit them with meaningful messages.
5. **Push and Pull Request:** Push your changes to your forked repository and submit a pull request.

## Reporting Bugs and Requesting Features
- Use the GitHub issue tracker to report bugs or request new features. Please be as detailed as possible when describing the issue.

# Development Setup

To set up a development environment for this project, follow these steps:

## Prerequisites
- [Node.js](https://nodejs.org/) (version X or higher)
- npm (Node package manager)

## Installation Steps
1. **Clone the Repository:**  
   `git clone https://github.com/edson-alive/Hostweb.git`
2. **Navigate to Directory:**  
   `cd Hostweb`
3. **Install Dependencies:**  
   `npm install`
4. **Run the Project:**  
   `npm start`

## Running Tests
- To run the tests, use the command: `npm test`

# Code Standards

Adhering to the following coding standards will ensure code quality and consistency:

## Code Formatting
- Use [Prettier](https://prettier.io/) for automatic code formatting.

## Naming Conventions
- Use camelCase for variable and function names.
- Use PascalCase for class names.

## Commit Message Conventions
- Use the imperative mood in commit messages (e.g., "Add feature" instead of "Added feature").
- Include a brief description of the changes made.
